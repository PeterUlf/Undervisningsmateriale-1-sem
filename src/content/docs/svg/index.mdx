## Hvorfor bruge SVG?

1. **Skalerbarhed:** Ser skarpt ud på alle skærme (mobil, retina, storskærm).
2. **Filstørrelse:** Ofte meget mindre end JPG/PNG for simple logoer, ikoner og illustrationer.
3. **Styling:** Kan styles med CSS (farve, størrelse, animation).
4. **Interaktivitet:** Kan manipuleres med JavaScript.

</Slide>

# Hvad er en SVG?

- **SVG** = Scalable Vector Graphics
- **Vektorbillede**, ikke pixels
- **Skalerbart** uden kvalitetstab
- **Tekstbaseret** (XML)

## Grundstruktur i en SVG

- `<svg>`: starttag med bredde/højde
- `<circle>`: en figur
- `xmlns`: navnerum (nødvendigt i standalone `.svg`-filer)

```svg
<svg xmlns="http://www.w3.org/2000/svg" width="100" height="100">
  <circle cx="50" cy="50" r="40" fill="blue" />
</svg>
```

![Eksempel på grundstruktur](/illustrationer/svg_grundstruktur.webp)

**Note:** `xmlns` kan udelades, hvis SVG'en er inline i HTML.

## Koordinatsystem i SVG

SVG bruger et koordinatsystem, hvor:

- **Origo (0,0)** er i **øverste venstre hjørne**
- **x-aksen** går mod højre (stigende værdier)
- **y-aksen** går nedad (stigende værdier)

```svg
<svg xmlns="http://www.w3.org/2000/svg" width="200" height="200">
  <circle cx="50" cy="50" r="20" fill="red" />
  <circle cx="150" cy="150" r="20" fill="blue" />
</svg>
```

- `cx` og `cy` = cirklens centrum (center x, center y)
- `r` = radius
- Rød cirkel: centrum ved (50, 50)
- Blå cirkel: centrum ved (150, 150)

## viewBox

Attributten `viewBox` styrer det **koordinatsystem**, din SVG tegnes i.

Den indeholder **4 værdier**:

```svg
viewBox="0 0 100 100"
```

![Eksempel på viewBox 0 0 40 40](/illustrationer/svg_viewbox.webp)

1. **Første tal:** venstre startpunkt på x-aksen (minX)
2. **Andet tal:** øverste startpunkt på y-aksen (minY)
3. **Tredje tal:** bredde på koordinatsystemet
4. **Fjerde tal:** højde på koordinatsystemet

### Eksempel:

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 200 200" width="100" height="100">
  <circle cx="100" cy="100" r="80" fill="purple" />
</svg>
```

Selvom SVG'en kun er 100x100 pixels, viser den et 200x200 koordinatsystem - cirklen bliver "zoomet ud".

</Slide>
<div class="page-break"></div>
<Slide>

## viewBox som "kamera"

`viewBox` fungerer lidt som et **kamera**, der bestemmer:

- hvilket udsnit af tegningen man ser
- hvor stort det vises
- hvordan det tilpasses, når SVG'en ændrer størrelse

**Tænk på det sådan:**

- SVG'ens `width` og `height` = hvor stor rammen er på skærmen
- `viewBox` = hvilket koordinatområde der vises i rammen

### Eksempel med zoom-effekt:

```svg
<!-- Normal visning: 1:1 -->
<svg width="100" height="100" viewBox="0 0 100 100">
  <circle cx="50" cy="50" r="40" fill="blue" />
</svg>

<!-- Zoomet ind: viser kun 50x50 område -->
<svg width="100" height="100" viewBox="0 0 50 50">
  <circle cx="25" cy="25" r="20" fill="red" />
</svg>
```

Den røde cirkel ser dobbelt så stor ud!

### Øvelse: Eksperimenter med viewBox

1. Lav en SVG med en cirkel: `width="200" height="200"` og `viewBox="0 0 200 200"`
2. Ændr viewBox til `"0 0 100 100"` - hvad sker der?
3. Prøv `viewBox="50 50 100 100"` - hvorfor ser du kun en del?

## Grundlæggende former

SVG har tags til forskellige former:

- `<rect>`: Rektangel
- `<circle>`: Cirkel
- `<ellipse>`: Ellipse
- `<line>`: Linje
- `<polyline>`: Linje med flere punkter
- `<polygon>`: Lukket form med flere sider

### Eksempel: Rektangel og linje

```svg
<svg xmlns="http://www.w3.org/2000/svg" width="200" height="200">
  <rect x="10" y="10" width="80" height="60" fill="green" />
  <line x1="100" y1="20" x2="180" y2="80" stroke="purple" stroke-width="3" />
</svg>
```

- `<rect>`: `x`, `y` = øverste venstre hjørne, `width`, `height` = størrelse
- `<line>`: `x1`, `y1` = startpunkt, `x2`, `y2` = slutpunkt

### Øvelse: Tegn former

2. Lav en linje med `<line>`

## Grupper med `<g>`

`<g>` samler flere elementer, så du kan styre dem samlet (fx samme farve, transform, opacity osv.).

```svg

  <circle cx="30" cy="30" r="10" />
  <rect x="50" y="20" width="20" height="20" />
</g>
```

## Transform

Med `transform` kan du rotere, skalere og flytte elementer:

- **`translate(x, y)`**: Flytter elementet
- **`rotate(grader)`**: Roterer elementet (omkring origo eller angivet punkt)
- **`scale(faktor)`**: Skalerer elementet (1 = normal, 2 = dobbelt størrelse)

### Eksempel: Rotation

```svg
<svg xmlns="http://www.w3.org/2000/svg" width="200" height="200">
  <!-- Original blå firkant -->
  <rect x="50" y="50" width="40" height="40" fill="blue" opacity="0.5" />
  <!-- Roteret rød firkant -->
  <rect x="50" y="50" width="40" height="40" fill="red"
        transform="rotate(45 70 70)" />
</svg>
```

Den røde firkant er roteret **45 grader** omkring punktet **(70, 70)** = centrum af firkanten.

**Vigtigt:** `rotate(45)` roterer omkring (0,0), men `rotate(45 70 70)` roterer omkring (70,70).

### Øvelse: Transform

1. Lav en figur og roter den 30 grader: `transform="rotate(30)"`

3. Kombiner transforms: `transform="translate(50,50) rotate(45) scale(1.5)"`
4. Hvad sker der, hvis du bytter om på rækkefølgen?

## SVG i webprojekter

Du kan bruge SVG på to måder:

### 1. Som et billede (`<img>`-tag)

✅ **Fordele:** Simpelt, cachebart, nemt at genbruge  
❌ **Ulemper:** Kan ikke styles med CSS eller manipuleres med JavaScript

```html
<img src="logo.svg" alt="Mit logo" width="100" />
```

![SVG som img-tag](/illustrationer/svg_i_web_1.webp)

### 2. Inline SVG (direkte i HTML)

✅ **Fordele:** Kan styles med CSS, interaktiv med JavaScript, ingen ekstra HTTP-request  
❌ **Ulemper:** Større HTML-fil, kan ikke caches separat

```html
<svg width="100" height="100">
  <circle cx="50" cy="50" r="40" fill="blue" />
</svg>
```

![SVG inline med xmlns](/illustrationer/svg_i_web_2.webp)

**Husk:** Standalone `.svg`-filer skal have `xmlns="http://www.w3.org/2000/svg"`, men inline SVG i HTML behøver det ikke.

1. Indsæt en cirkel-SVG direkte i `oevelse.html` lige under `<h1>`.
2. Tilføj samme SVG som en fil og brug `<img src="cirkel.svg">`

## Styling med CSS

Når du bruger inline SVG, kan du style det med CSS:

```html
<svg width="100" height="100">
  <circle class="my-circle" cx="50" cy="50" r="40" />
</svg>
```

```css
.my-circle {
  fill: red;
  stroke: black;
  stroke-width: 2;
}

.my-circle:hover {
  fill: blue;
  cursor: pointer;
}
```

**Note:** Dette virker kun med inline SVG, ikke med `<img>`-tag.

## Opsummering: Når skal du bruge hvad?

### Brug `<img src="ikon.svg">`:

- ✅ Logoer der ikke skal ændres
- ✅ Dekorative illustrationer
- ✅ Når du vil cachning og hurtig load

### Brug inline `<svg>`:

- ✅ Ikoner der skal ændre farve (fx menu-ikoner)
- ✅ Interaktive elementer (klikbare dele)
- ✅ Animationer med CSS/JavaScript
- ✅ Når du vil undgå ekstra HTTP-requests

### Brug `viewBox`:

- ✅ Når SVG'en skal være responsiv
- ✅ Når du vil zoome ind/ud på indhold
- ✅ Når du vil sikre korrekt skalering

**Pro tip:** Brug altid `viewBox` i dine SVG'er for bedre responsivitet!
