## Hvad er en komponent i Astro?

En **komponent** er et lille, genbrugeligt stykke UI – fx en knap, et kort eller et helt layout.

I Astro er komponenter som regel filer med endelsen `.astro` i mappen `src/components/`.

```text
src/
  components/
    Footer.astro
    Header.astro
    ServiceCard.astro

```

</Slide>

<div class="page-break"></div>

<Slide>

## Grundstruktur i en `.astro`-fil

En Astro-komponent består af:

1. **Frontmatter** (mellem `---`), hvor du kan importere ting og definere props
2. **Template** (HTML + JSX-lignende syntaks)

Ensempel på en card komponent, som består af frontmatter, noget almindeligt html og nogle props som bliver sat ind:

Card.astro

```astro
---
const { title, body } = Astro.props;
---
<article class="card">
  <h2>{title}</h2>
  <p>{body}</p>
</article>

```

</Slide>

<div class="page-break"></div>

<Slide>

Her er så et eksempel hvor komponenten også indeholder styling i bunden af filen:

Card.astro (her med styling)

```astro
---
const { title, body } = Astro.props;
---
<article class="card">
  <h2>{title}</h2>
  <p>{body}</p>
</article>

<style>
  .card {
    background: #5000ca;
    border-radius: 8px;
    padding: 1rem;
    margin-block: 1rem;
  }

  h2 {
    color: #fff;
  }

  p {
    color: #cecece;
  }
</style>
```

- `Astro.props` indeholder de værdier du sender ind, når du bruger komponenten.
- `{her-står-navnet-på-en-prop}` er alt det indhold, du putter mellem åbnings- og lukketagget.

</Slide>

## Brug af en komponent

Hvis du har en `Card.astro`, kan du bruge den i en side eller en anden komponent:

```astro
---
import Card from "../components/Card.astro"
---
--HER MANGLE EN MASSE STANDARD HTML KODE--

      <section>
        <div class="cards">
          <Card title="Kort 1 er her" body="Indhold for kort 1." />
          <Card title="Også kort 2, men ikke 3" body="Indhold for kort 2." />
        </div>

--HER MANGLE EN MASSE STANDARD HTML KODE--
```

Komponenten bliver oversat til almindelig HTML i det færdige build.

## Slots – fleksible pladser til indhold

I stedet for at bruge `body` som en prop, kan du bruge **slots** til at sende HTML-indhold ind i komponenten.

Her er en InfoBox.astro komponent, der bruger slots til svaret:

```astro
---
const { question } = Astro.props;
---
<aside class="infobox">
  <h3>{question}</h3>
  <slot />
</aside>

<style>
  .infobox {
    background: #e0f2fe;
    border-left: 4px solid #0284c7;
    border-radius: 4px;
    padding: 1.5rem;
    margin-block: 1rem;
  }

  h3 {
    color: #0284c7;
    margin-top: 0;
  }
</style>
```

### Sådan bruges slots

```astro
<InfoBox question="Hvad er en komponent?">
  <p>En komponent er et <strong>genbrugeligt stykke UI</strong>, som du kan bruge flere steder i dit projekt.</p>
  <p>Det gør din kode mere organiseret og nemmere at vedligeholde.</p>
</InfoBox>
```

**Hvordan virker det?**

1. `question="Hvad er en komponent?"` bliver sendt til komponenten som en prop og indsættes i `<h3>{question}</h3>`
2. Alt HTML mellem `<InfoBox>` og `</InfoBox>` (paragraferne med svaret) bliver indsat hvor `<slot />` står i komponenten
3. Resultatet bliver til almindelig HTML:

```html
<aside class="infobox">
  <h3>Hvad er en komponent?</h3>
  <p>
    En komponent er et <strong>genbrugeligt stykke UI</strong>, som du kan bruge
    flere steder i dit projekt.
  </p>
  <p>Det gør din kode mere organiseret og nemmere at vedligeholde.</p>
</aside>
```

**Fordelen ved slots:**

Med `<slot />` kan du indsætte komplekst HTML-indhold, andre komponenter, billeder, lister osv. – i stedet for kun simpel tekst via props.
